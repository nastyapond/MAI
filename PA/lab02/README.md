# Виртуальный датчик для контроля процесса обжига в печи

## Описание
Данный проект реализует виртуальный датчик для прогнозирования концентрации выходного продукта в процессе обжига сырья на промышленном предприятии. Модель использует данные телеметрии печи для предсказания лабораторных замеров концентрации в реальном времени, компенсируя задержку лабораторного анализа.

## Структура
- `lab02.ipynb` - Jupyter Notebook с полным пайплайном решения
- `requirements.txt` - зависимости
- `data_train.csv` - обучающие данные телеметрии (минутные измерения 16 показателей телеметрии за 8 месяцев для обучения)
- `target_train.csv` - обучающие данные лабораторных замеров (лабораторные замеры концентрации целевого продукта (нерегулярные, с задержкой 10-15 мин))
- `data_test_small.csv` - тестовые данные телеметрии
- `target_test_small.csv` - тестовые данные лабораторных замеров

## Запуск
1. Установить окружение и зависимости:
```bash
cd lab02
pip install -r requirements.txt
```
2. Запустить Jupyter Notebook:
```bash
jupyter notebook lab02.ipynb
```
3. Выполнить ячейки последовательно.
4. Для проверки на тестовом наборе использовать ячейку с синхронизацией `data_test_small.csv` и `target_test_small.csv`.

## Ход работы
- Выполнен EDA: анализ пропусков, выбросов, распределений и частоты лабораторных замеров.
- Синхронизированы минутные телеметрия и нерегулярные лабораторные замеры с учётом задержки анализа.
- Созданы временные признаки: лаги, скользящие статистики, производные и темпы роста.
- Построены три модели: XGBoost, Random Forest и Linear Regression.
- Оценена точность моделей с помощью MAE, RMSE, MAPE и Directional Accuracy.
- Проведён анализ остатков и важности признаков для интерпретации результатов.

## Результаты
Модель XGBoost достигает следующих метрик на тестовых данных:
- MAE: 0.0658
- RMSE: 0.0850
- MAPE: 0.2900
- Directional Accuracy: 0.6487

## Интерпретация результатов
- Низкие значения MAE и RMSE показывают, что модель предсказывает концентрацию достаточно точно для оперативного контроля процесса.
- MAPE ~0.29 означает, что средняя относительная ошибка прогноза составляет примерно 29%.
- Tочность направления изменений (DA) 0.6487 показывает, что модель в 65% случаев правильно определяет направление изменения концентрации, что важно для быстрого реагирования операторов.
- Важные признаки - лаговые значения `telemetry_0` и скользящие статистики. История изменений телеметрии сильнее всего связана с качеством продукта.
