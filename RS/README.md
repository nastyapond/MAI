# VK-LSVD Recommendation System

Система рекомендации видеоклипов для пользователей на основе коллаборативной фильтрации.

## Описание

Решение использует **Alternating Least Squares (ALS)** - эффективный алгоритм для матричной факторизации в задачах коллаборативной фильтрации.

### Ключевые компоненты:

1. **Построение матрицы взаимодействий (Interaction Score)**
   - Like: вес 5.0 (сильное положительное взаимодействие)
   - Bookmark: вес 4.0 (сохранение видео)
   - Share: вес 3.0 (поделиться)
   - Click on author: вес 2.0 (интерес к автору)
   - Open comments: вес 1.0 (слабое взаимодействие)
   - Timespent: нормализованное время просмотра
   - Dislike: штраф (умножение на 0.2)

2. **Матричная факторизация (ALS)**
   - Разложение матрицы взаимодействий на две низкорранговые матрицы
   - Минимизация: ||X - UV^T||_F^2 + λ(||U||_F^2 + ||V||_F^2)
   - Итеративное решение нормальных уравнений

3. **Confidence Weighting**
   - Применение коэффициента уверенности (alpha) к матрице взаимодействий
   - Усиливает влияние явных взаимодействий

## Использование

### Базовое использование:
```bash
python vk_recs_als.py --input train.parquet --output recommendations.csv
```

### С параметрами:
```bash
python vk_recs_als.py \
  --input train.parquet \
  --output recommendations.csv \
  --n_recs 10 \
  --factors 64 \
  --regularization 0.01 \
  --iterations 20 \
  --alpha 40.0 \
  --validate true \
  --seed 42
```

### Параметры:

- `--input` (обязательно): путь к parquet файлу или папке с parquet файлами
- `--output` (обязательно): путь для сохранения CSV с рекомендациями
- `--n_recs`: количество рекомендаций на пользователя (по умолчанию 10)
- `--factors`: количество скрытых факторов в ALS (по умолчанию 64)
- `--regularization`: коэффициент регуляризации (по умолчанию 0.01)
- `--iterations`: количество итераций ALS (по умолчанию 20)
- `--alpha`: коэффициент уверенности (по умолчанию 40.0)
- `--validate`: проводить ли валидацию (по умолчанию true)
- `--seed`: seed для воспроизводимости (по умолчанию 42)

## Формат данных

### Входные данные (Parquet)
Таблица взаимодействия user-item:
- `user_id` (uint32): идентификатор пользователя
- `item_id` (uint32): идентификатор видео
- `timespent` (uint8): время просмотра в секундах (0-255)
- `like` (boolean): пользователь лайкнул видео
- `dislike` (boolean): пользователь дизлайкнул видео
- `share` (boolean): пользователь поделился видео
- `bookmark` (boolean): пользователь сохранил видео
- `click_on_author` (boolean): пользователь открыл страницу автора
- `open_comments` (boolean): пользователь открыл комментарии

### Выходные данные (CSV)
```
user_id,recs
123,456
123,789
123,101112
124,456
...
```

Каждая строка содержит один `user_id` и один рекомендованный `item_id`.
Рекомендации отсортированы по релевантности (сначала наиболее релевантные).

## Метрика оценки

**MAP@10** (Mean Average Precision at 10)

```
MAP@10 = (1/|U|) * Σ AP@10(u)

где AP@10(u) = (1/min(m, 10)) * Σ P(k) * rel(k)
```

- `P(k)` - precision at position k
- `rel(k)` - релевантность на позиции k (1 если item в ground truth, 0 иначе)
- `m` - количество релевантных items для пользователя u

## Особенности реализации

### Преимущества ALS:
1. **Эффективность**: O(nnz + k²(n + m)) на итерацию (где k - число факторов, nnz - число ненулевых элементов)
2. **Параллелизуемость**: каждый фактор пользователя/предмета может обновляться независимо
3. **Сходимость**: гарантированная монотонная сходимость
4. **Интерпретируемость**: факторы имеют смысл в скрытом пространстве

### Оптимизация:
- Использование разреженных матриц (scipy.sparse)
- Прямое решение нормальных уравнений (numpy.linalg.solve)
- Фильтрация уже просмотренных видео при рекомендации

## Сравнение с baseline

Baseline использует простую эвристику: рекомендует топ-10 популярных видео всем пользователям.

ALS дает персонализированные рекомендации на основе истории пользователя, что обычно приводит к значительному улучшению MAP@10.

## Примечания

- Валидация проводится по схеме leave-one-out на тренировочном наборе
- Для каждого пользователя одно случайное взаимодействие оставляется в тестовом наборе
- Рекомендации генерируются только для пользователей, присутствующих в тренировочном наборе
- Гарантируется, что рекомендации не включают видео, с которыми пользователь уже взаимодействовал
